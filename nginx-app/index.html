O que é o Azure Container Apps?

Azure Container Apps (ACA) é um serviço gerenciado que executa containers Docker sobre Kubernetes, porém escondendo totalmente o cluster do usuário.

Você não gerencia nodes, pools, upgrades, CNI ou control plane. O foco passa a ser exclusivamente a aplicação, seus eventos e sua escala.

  • • Execução baseada em eventos e requisições HTTP
  • • Escala automática (KEDA)
  • • Billing por vCPU, memória e tempo de execução
  • • Integração nativa com Log Analytics e Dapr

Arquitetura Conceitual

Environment

Boundary lógico que agrupa Container Apps, redes, observabilidade e componentes compartilhados como Dapr.

Container App

Unidade de deploy. Define imagem, recursos, regras de escala, ingress e variáveis de ambiente.

Revisions

Modelo de versionamento imutável que permite blue/green, canary e rollback controlado.

Quando usar (e quando evitar)

Use Container Apps quando

  • • Microsserviços HTTP ou event-driven
  • • Workloads com tráfego variável
  • • Necessidade de escalar até zero
  • • Time não quer operar Kubernetes

Evite Container Apps quando

  • • Controle fino de rede (CNI, Calico, Istio)
  • • Workloads stateful complexos
  • • Dependência de CRDs específicos
  • • Necessidade de acesso direto ao cluster
